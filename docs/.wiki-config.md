# Wiki-Konfiguration für Azure DevOps

Diese Dokumentation ist für die Verwendung als Azure DevOps Wiki optimiert.

## Struktur

Die Dokumentation ist in thematische Ordner unterteilt:

- **Getting-Started** - Erste Schritte und Installation
- **Architecture** - Systemarchitektur und Komponenten
- **Configuration** - Konfigurationsverwaltung
- **Setup-Wizard** - Installationsassistent
- **Security** - Sicherheit und Authentifizierung
- **Operations** - Betrieb und Wartung
- **Development** - Entwicklungsumgebung
- **CI-CD** - Continuous Integration & Deployment
- **Reference** - Technische Referenz
- **Roadmap** - Zukunftspläne

## Navigation

Die Navigation wird über `.order` Dateien gesteuert. Jeder Ordner enthält eine `.order` Datei, die die Reihenfolge der Seiten definiert.

## Azure DevOps Wiki Setup

### Als Wiki publizieren

1. Gehe zu **Azure DevOps** → **Overview** → **Wiki**
2. Wähle **Publish code as wiki**
3. Wähle das Repository und den Branch
4. Setze den Folder auf `/docs`
5. Benenne das Wiki (z.B. "ReadyStackGo Documentation")

### Wiki-Features

- ✅ Automatische Navigation über `.order` Dateien
- ✅ Suchfunktion über alle Seiten
- ✅ Versionierung über Git
- ✅ Markdown-Rendering
- ✅ Mermaid-Diagramme (in Architecture)
- ✅ Code-Syntax-Highlighting

## Lokale Vorschau

Zum lokalen Testen der Dokumentation:

```bash
# Mit einem Markdown-Previewer
code docs/index.md

# Oder mit einem Static Site Generator
npx docsify serve docs
```

## Wartung

### Neue Seite hinzufügen

1. Erstelle die `.md` Datei im entsprechenden Ordner
2. Füge den Dateinamen (ohne `.md`) zur `.order` Datei hinzu
3. Verlinke die Seite im `index.md` wenn relevant

### Seite umbenennen

1. Benenne die Datei um
2. Aktualisiere den Eintrag in der `.order` Datei
3. Aktualisiere alle internen Links

### Ordner hinzufügen

1. Erstelle den Ordner
2. Erstelle eine `.order` Datei im Ordner
3. Füge den Ordnernamen zur Root `.order` Datei hinzu
4. Erstelle eine Overview-Seite im Ordner

## Best Practices

- **Klare Titel**: Jede Seite sollte einen H1-Titel haben
- **Interne Links**: Verwende relative Pfade für Links
- **Bilder**: Speichere Bilder in `assets/images/`
- **Diagramme**: Bevorzuge Mermaid für Diagramme (als Code)
- **Konsistenz**: Halte den Schreibstil konsistent
